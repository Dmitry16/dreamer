# MVP Specification

## Goal

Deliver a minimum lovable product that enables a complete Jungian dreamwork session from dream capture to reflective integration, without overinterpretation or therapeutic claims.

---

## Core MVP Features

### 1. Dream Capture
- Free-text dream input
- Date/time (auto-filled, editable)
- Optional context: waking mood, life context

### 2. Dream Structuring
Automatically extract and present editable elements:
- Characters
- Symbols / objects
- Settings / locations
- Actions / events
- Emotions
- Narrative shifts ("the turn")
- Ending image

### 3. Personal Associations
For each symbol:
- Free-text association prompts
- Emotional valence (positive / negative / mixed)
- Salience rating (how important it feels)

### 4. Interpretive Hypotheses
Generate 2–3 hypotheses grounded in:
- Dream evidence
- User associations
- Jungian lenses (compensation, shadow, archetypal dynamics)

Each hypothesis includes:
- Supporting observations
- Interpretive claim
- Reflective test question
- Dismiss / accept feedback

### 5. Integration Practice
- 1–2 reflective questions
- 1 small behavioral or attitudinal adjustment suggestion
- Optional journaling prompt

### 6. Dream History
- Chronological list of past dreams
- View previous interpretations
- No cross-dream analysis in MVP

---

## Explicit Non-Goals (MVP)
- No active imagination
- No archetype labeling UI
- No social or sharing features
- No diagnosis, prediction, or advice
